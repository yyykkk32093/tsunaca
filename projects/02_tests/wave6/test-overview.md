# Wave6 テスト戦略

> **最終更新**: 2026-04-27
> **対象 Phase**: Phase 7（ヘルプ）/ Phase 8（問い合わせ + SystemAdmin 基盤 + 最小管理画面）/ Phase 9（ハンバーガーメニュー）
> **要件参照**: [overview.md](../../00_requirements/202603_07_bugfix-and-refactoring_wave6/overview.md)

## テスト戦略サマリ

Wave6 Ph7-9 は同時実装のため、**共通基盤（AppLayout 再設計、SystemAdmin 認可）が他 Phase の前提**となる。テストも次の3層に分けて実施する：

1. **共通基盤層**: 認可境界、AppLayout シェル、ルーティング保護
2. **機能層**: 各 Phase 固有の機能要件
3. **統合層**: Phase 間の導線・データ連携・権限別表示

## 横断観点

### 認可境界（Wave6 で最重要）
| 観点                                                                          | 該当Phase |
| ----------------------------------------------------------------------------- | --------- |
| 未認証ユーザーが `/help`（public 記事）を閲覧できる                           | Ph7       |
| 未認証ユーザーが `/help/admin/*` 記事を閲覧できない                           | Ph7       |
| 未認証ユーザーが `/contact` にアクセスするとログインへ誘導                    | Ph8       |
| `/contact/anonymous` は未認証でアクセス可（reCAPTCHA 必須）                   | Ph8       |
| 一般ユーザー（USER）が `/admin/*` にアクセスすると 404                        | Ph8-A     |
| OPERATOR が `/admin/inquiries` を閲覧/操作できる                              | Ph8-C     |
| ハンバーガーメニュー「運営メニュー」セクションが SystemAdmin のみに表示される | Ph9       |
| 一般ユーザーのメニューに admin 系項目が一切表示されない                       | Ph9       |

### a11y（Q18 完全対応）
| 観点                                          | 該当Phase         |
| --------------------------------------------- | ----------------- |
| キーボードのみで全機能操作可能                | Ph7/Ph8/Ph9       |
| フォーカストラップ（Sheet/Dropdown/Dialog）   | Ph9, Ph8 フォーム |
| ARIA 属性が正しく付与される                   | 全 Phase          |
| Esc でモーダル/メニューがクローズ             | Ph9, Ph8          |
| `prefers-reduced-motion` でアニメーション抑制 | Ph9               |
| スクリーンリーダーで状態読み上げ              | 全 Phase          |

### i18n
| 観点                                                  | 該当Phase |
| ----------------------------------------------------- | --------- |
| ヘルプ Markdown が ja/en ロケール別に切り替わる       | Ph7       |
| InquiryCategory の labelI18n が UI で正しく表示される | Ph8       |
| メニュー項目ラベルが i18n 対応                        | Ph9       |

### セキュリティ
| 観点                                                         | 該当Phase |
| ------------------------------------------------------------ | --------- |
| 添付ファイルのウイルススキャン（INFECTED は拒否）            | Ph8       |
| レート制限（IP + ユーザー単位）                              | Ph8       |
| reCAPTCHA トークン検証                                       | Ph8       |
| SystemAdmin 認可ミドルウェアのバイパス不可                   | Ph8-A     |
| Markdown レンダラ XSS 対策（dangerouslySetInnerHTML 不使用） | Ph7       |

### 性能
| 観点                                                   | 該当Phase |
| ------------------------------------------------------ | --------- |
| ヘルプ全文検索のクライアント処理が体感 100ms 以内      | Ph7       |
| 添付スキャン中のフォーム送信ブロックが UX を損なわない | Ph8       |
