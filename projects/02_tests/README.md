# テスト観点ドキュメント運用

> 各Wave/Phaseの実装に対するテスト観点を集約するディレクトリ。
> 自動テスト（unit/integration/e2e）と手動受入テストの両方を観点ベースで管理する。

## ディレクトリ構成

```
projects/02_tests/
├── README.md                  ← 本ファイル（運用ガイド）
├── _template.md               ← Phase別テスト観点ファイルのテンプレート
└── wave{N}/                   ← Wave単位
    ├── test-overview.md       ← Wave全体のテスト戦略・横断観点
    └── phase{M}-tests.md      ← Phase別テスト観点
```

## 作成タイミング

- 各 Phase の **要件確定時**（実装着手前）にテンプレートをコピーして観点を起票する
- 実装中に観点が追加・変更された場合は随時更新する
- Phase 完了時に「実施結果」列を埋めてレビュー対象とする

## 観点表のフォーマット

各 phase{M}-tests.md は以下の表形式で管理する：

| ID  | 観点カテゴリ | 観点 | 種別 | 優先度 | 実施結果 | 備考 |
| --- | ------------ | ---- | ---- | ------ | -------- | ---- |

- **観点カテゴリ**: 機能 / 認可 / バリデーション / a11y / 性能 / セキュリティ / i18n 等
- **種別**: `auto-unit` / `auto-integration` / `auto-e2e` / `manual` / `manual-探索的`
- **優先度**: P0（必須）/ P1（重要）/ P2（任意）
- **実施結果**: ❌ 未実施 / 🚧 進行中 / ✅ 合格 / ⚠️ 不合格

## 自動テスト配置との対応

| 種別               | 配置先                                             |
| ------------------ | -------------------------------------------------- |
| `auto-unit`        | `backend/test/` または `frontend/src/**/*.test.ts` |
| `auto-integration` | `backend/test/integration/`                        |
| `auto-e2e`         | `backend/test/e2e/`                                |
| `manual`           | 本ドキュメントのみで管理（実施結果列を更新）       |
